# الامتثال القانوني - PDPL و SAMA

## نظرة عامة

تطبيق **نهج** مصمم ليكون متوافقاً بالكامل مع:

1. **نظام حماية البيانات الشخصية (PDPL)** — المملكة العربية السعودية
2. **متطلبات البنك المركزي السعودي (SAMA)**

---

## أولاً: PDPL (نظام حماية البيانات الشخصية)

### المادة 5 — مبادئ معالجة البيانات

| المبدأ | كيفية التطبيق |
|--------|--------------|
| **المشروعية** | الموافقة الصريحة عند التسجيل (`consent: true`) |
| **الغرض المحدد** | تُستخدم البيانات فقط لخدمات الإدارة المالية |
| **التناسب** | نجمع الحد الأدنى من البيانات (Data Minimization) |
| **الدقة** | المستخدم يتحكم في بياناته ويحدّثها |
| **التحديد الزمني** | حذف تلقائي بعد انتهاء صلاحية الاحتفاظ (TTL) |

### المادة 12 — الإفصاح

- ✅ سياسة الخصوصية واضحة قبل التسجيل
- ✅ يُحفظ timestamp للموافقة في قاعدة البيانات
- ✅ لا مشاركة مع أطراف ثالثة بدون إذن

### المادة 18 — حق الحذف

```typescript
// حذف ناعم (Soft Delete) - يحفظ سجل المراجعة
await deleteUser(userId);  // يُغير status إلى 'deleted'
```

### المادة 21 — أمن البيانات

| الضمانة | التقنية المستخدمة |
|---------|-----------------|
| التشفير في وضع الراحة | AWS KMS (AES-256) |
| التشفير أثناء النقل | HTTPS/TLS 1.3 |
| التحكم في الوصول | IAM Roles + Least Privilege |
| سجل المراجعة | جدول `audit-logs` بـ DynamoDB |

### المادة 29 — إقامة البيانات

```
✅ كل البيانات في: me-central-1 (الرياض)
✅ لا نقل خارج المملكة
✅ DynamoDB + Lambda في نفس المنطقة
```

## ثانياً: SAMA (البنك المركزي السعودي)

### متطلبات إطار الأمن السيبراني

| المتطلب | الحالة | التفاصيل |
|---------|--------|---------|
| **تشفير البيانات** | ✅ | KMS AES-256 |
| **التحكم في الوصول** | ✅ | IAM + Role-Based |
| **سجلات الأحداث** | ✅ | Audit Logs 7 سنوات |
| **اختبار الاختراق** | ⏳ | مطلوب قبل الإنتاج |
| **إدارة الثغرات** | ✅ | npm audit + CodeQL |

## قائمة التحقق (Checklist)

### قبل الإطلاق (Production)

- [ ] تدقيق أمني خارجي (Penetration Testing)
- [ ] مراجعة سياسة الخصوصية من محامٍ متخصص
- [ ] اختبار تشفير KMS
- [ ] التحقق من عدم نقل بيانات خارج me-central-1
- [ ] اختبار عملية حذف بيانات المستخدم (PDPL Art. 18)
- [ ] تفعيل AWS CloudTrail للتدقيق الكامل
- [ ] تفعيل DynamoDB Point-in-Time Recovery
- [ ] اختبار آلية الـ Audit Logs
- [ ] إعداد تنبيهات CloudWatch للأحداث الأمنية
- [ ] توثيق جميع معالجات البيانات (Data Processing Register)

### الصيانة المستمرة

- [ ] مراجعة شهرية لسجلات الوصول
- [ ] تحديث نصف سنوي لسياسة الخصوصية
- [ ] اختبار سنوي للتعافي من الكوارث
- [ ] تحديث دوري لمفاتيح KMS (Key Rotation مفعّل)
- [ ] مراجعة ربع سنوية للصلاحيات
